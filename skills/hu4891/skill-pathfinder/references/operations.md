# 执行标准与行为准则 (Execution Operations)

## 0. 绝对铁律：双重防线执行锁 (The Execution Lock)
在处理所有**外部资源引入**的任务时（特别是涉及 `npx` 运行时拉取、`npm install`、`git clone` 或直接向硬盘写入新文件），**大模型严禁在没有用户授权的情况下私自发起终端长连接下载**！
- 任何形式的代码拉取指令在输入终端前，必须通过对话框或确认卡片向用户展示完整命令。
- 只有接收到用户明确的“确认/执行/Yes”反馈后，才能向后台发送执行指令。
- 这一规定是不可触碰的安全红线，防止形成任意代码执行 (RCE) 后门。

## 1. 原生方案优先生态 (Native Tools First)

本模块旨在让用户的 Agent 持续拥有最新的硬核生产力扩展，同时保持极高的推荐纯净度与交互优雅。

## 1. 甄选目标与黄金准则 (The Target Standard)
大模型在考虑向用户推荐或主动拉取任何新 Skill 之前，**必须在脑海中对目标进行以下严苛的标尺测算**：
> 值得被打扰并推荐给用户的组件，必须具有明显的**“生产力突变”**、能大幅扩展现有 Agent 的**物理作业边界**（如：直接操控本地文件系统、能链接或控制外部重型 API、打通复杂工作流）。
> ❌ **绝对排斥**：纯文字游戏、算命娱乐、简单的套壳 API、重造的轮子。

## 2. 互补型全网多渠道雷达 (Multi-channel Discovery)
系统不再被动等待，而是定期或在用户表现出“探索新玩意”意图时，启用双线并行的嗅探机制：
- **前沿阵地（极客风暴）**：使用内置的搜索/联网能力，前往 **Twitter (X)** 等高端技术社群，检索 `#AI #AgentSkills #ClawHub #MCP` 等相关话题标签，捕获讨论度与热度正在飙升的前哨级实验性工具。
- **正规军火库（市场榜单）**：前往 **ClawHub、MCP Market、Smithery** 等核心白名单源站，抓取官方精选或近期下载量突然跃升的稳定版刚需组件。

这两个渠道互为补充。但所有被抓上来的“嫌疑名单”，都必须经过上述安全、高生产力的“准则过滤”和 `AgentGuard` 的底包安检后，才能脱颖而出。

## 3. UI 交互：纯净段落式推荐陈列 (Clean Text-only Formatting)
当经过层层筛选，你决定向用户展示你的“心动嘉宾”时，存在一条不可逾越的红线排版铁律：
🚨 **【严禁使用 Markdown 表格排版输出】** 🚨

系统与用户汇报推荐雷达成果时，必须使用清爽、利落的“一物一段”形式。每推荐一个组件，必须严格遵循此格式：

**【动态数据可视化 Widget】**，由这哥们在社区开源分享的设计。它能让你干巴巴的财务报表在终端直接动起来，非常适合你经常要做的销售额对比分析（突变生产力）。
🔗 直达源：`[https://clawhub.ai/xxx]`

*(每个组件空一行，结构为：加粗名字 + 1到2句话的人话痛点说明 + 短平快的直连。让阅读毫无负担。)*
