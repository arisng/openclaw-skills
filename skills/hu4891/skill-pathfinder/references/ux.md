# 用户交互与兜底保障 (UX & Fallback)

确保整个“调度-寻找-安装-执行”的精密链路对用户是友好的、透明的，以及最重要的——**受控且匹配用户语言环境的**。

## 1. 全局多语言适配机制 (Language Mirroring)
本元技能在对用户进行所有的话术反馈（包括状态栏、告警提示、询问授权）时，**绝对禁止硬编码为固定的中文或英文**。
- **自动镜像**：必须侦测并跟随用户在本次会话中发起的主控语言进行全向同步。例如，如果用户用英文写 “Find a tool”，你所有的进度更新和询问词都必须无缝切换为英文；若用户说中/日/韩文，同理必须跟从。
- **双语 / Emoji 警示辅助**：凡是进入诸如拦截等待、报错、系统级降级的极端状态时，推荐混合使用国际通用 Emoji（如 ⚠️, 🚨, 🌐, ✅, 📥）来打破语言隔阂加强视觉权重。

## 2. 状态透明与全景排期反馈
在执行复杂的编排链路时，如果预期静默时间较长，必须利用大模型的输出流通道，向用户反馈前置或当前的运行进程状态：
- **【状态同步】下达多任务规划流**：当拆解复杂意图后，无需等待，开场即明确告诉用户计划。
  *范例（依用户语言而定）*：“✅ **任务编排已确立 (Task Pipeline)**：`[查找股市研报 Skill]` -> `[财务图表生成 Skill]` -> `[总结并抄送邮件 Skill]`。开始执行第一步...”
- **【状态同步】向外搜索探寻中**：
  *范例（依用户语言而定）*：“🌐 **外网检索中 (Searching Globally)**：本地库缺失满足条件的功能，系统正在接入 ClawHub / MCP Market 进行外向寻源提取...”
> 🛡️ **安全规范要求**：针对 AgentGuard 这类底层内部核心安全审计动作，为了消除执行黑盒并在长期等待中给用户安全感，你**必须向用户明示当前进度**！
> 举例：“*我已找到 `excel-parser` 组件，正在为您进行深度的本地运行沙箱安全扫描，请稍候...*” 绝不允许在不打招呼的情况下私自拉取并在后台黑盒静默扫描。
## 3. 强核准拦截点 (Authorization Checkpoints)
为了防止失控，系统在特定节点必须触发强硬打断（Pause-and-Wait），绝对不允许先斩后奏：
- **【安装授权】环境短板补齐诊断完成时**：当 `diagnostics.md` 中的基线扫描探测到缺少了推荐技能（比如联网组件）并给出了官方白名单时。
- **【安装授权】从外部获取到全新技能库时**：当 `discovery.md` 寻找且底层通过了安全过滤，准备向本地部署一个未知的扩展库之前，必须提交方案对比报表并挂起。
**在这两个节点，你必须完全停下手头所有的下载/部署脚本，明确要求：“您同意我执行下载与挂载安装吗？(Do you allow me to proceed with installation?)”，在获得用户的显性许可（Sure/可以/同意/Yes）前，指令绝对锁死。**

## 4. 多意图消歧追问
- 用户的语言很多时候是不精确的。当需求所指不明，或系统检索到2个以上完全平级、能力重叠且无法取舍的 Skill 时，不允许系统瞎猜盲目强行执行。
- 应当主动打断，给出具体的选项让用户做选择题。
   - *“您刚才所说的‘帮我发那份报告’，是想调用【内网OA投递技能】还是调用【外部邮件抄送技能】发送呢？”*

## 3. 分阶段优雅降级 (Graceful Degradation)
如果整个流程走到死胡同：
1. 本地没有可用技能。
2. 跨网搜索也没有任何符合相关意图的结果，或者被安全扫描模块拦截掉了所有潜在可能。
3. （或）搜索到了选项，但发给用户确认后遭到了用户的无情拒绝。

**此时，请做到优雅降级：**
- 切勿直接生硬报错并崩溃。
- 礼貌地退回到系统最基础的原生对话（大语言）模式尝试给出一个大概的探讨结果。
- **记录落空需求 (Demand-Miss)**：将本次寻找和满足需求失败的过程整理进系统的日志中（例如一个无形的 Demand-Miss 池），将其作为未来更新系统底层官方技能的重要需求挖掘线索。
